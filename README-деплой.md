# Бэкенд «knit-charts» — запуск на Railway

Это сервер (Node + Express) для регистрации по email и облачного хранения проектов в Postgres.
Файлы: `server.js`, `package.json`, `.gitignore`.

## 1. Залить код на GitHub
Создай НОВЫЙ репозиторий, например `knit-charts-backend`, и загрузи в него эти три файла
(`server.js`, `package.json`, `.gitignore`) — в корень. Не клади внутрь лишней папки.

> Можно и в тот же репозиторий, но отдельный проще.

## 2. Создать сервис на Railway
1. Открой свой проект на Railway (там, где уже стоит **Postgres**).
2. **+ Add → GitHub Repo →** выбери `knit-charts-backend`.
3. Railway сам поймёт, что это Node, и запустит `npm install` + `npm start`.

## 3. Переменные окружения (Variables) у нового сервиса
Открой сервис бэкенда → вкладка **Variables** и добавь:
- `DATABASE_URL` → нажми **Add Reference / Reference Variable** и выбери из сервиса **Postgres** переменную `DATABASE_URL`.
  (Так бэкенд подключится к твоей базе. Если ссылку добавить нельзя — скопируй значение `DATABASE_URL` из Postgres вручную, и тогда добавь ещё `PGSSL=true`.)
- `JWT_SECRET` → любая длинная случайная строка (например 30+ символов). Никому не показывай.

После сохранения сервис перезапустится и сам создаст таблицы.

## 4. Получить публичный адрес
Сервис бэкенда → **Settings → Networking → Generate Domain**.
Получишь адрес вида `https://knit-charts-backend-production.up.railway.app`.
Проверь: открой `этот-адрес/health` — должно показать `{"ok":true}`.

## 5. Подключить сайт
1. Открой свой сайт (knit-charts.vercel.app).
2. Вверху справа нажми **«Войти»**.
3. В поле **«Адрес сервера»** вставь адрес с Railway (без / в конце).
4. Введи email и пароль (от 6 символов) → **«Регистрация»**.
5. Готово. Теперь в разделе «Мои проекты» появятся кнопки **☁ В облако** и **☁ Из облака** —
   проекты сохраняются на сервер и доступны с любого устройства после входа.

## Примечания
- CORS открыт, поэтому сайт с Vercel свободно обращается к серверу.
- Пароли хранятся в зашифрованном виде (bcrypt). Подтверждение email пока не требуется.
- Бесплатных ресурсов Railway хватает для старта; при росте — платный план.
- Старый сервис «knit-charts» на Railway (если это статический сайт) не нужен — сайт живёт на Vercel. Можно оставить или удалить.
